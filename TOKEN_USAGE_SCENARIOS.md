# 토큰 사용 시나리오별 동작 방식

## 시나리오별 자동 접속 동작

### 1. 매일 사용하는 사용자

```
Day 1: 회원가입/로그인
  → Access Token (15분) + Refresh Token (15일) 발급

Day 2-15: 매일 접속
  → Access Token 만료 감지
  → Refresh Token으로 자동 갱신
  → ✅ 자동 로그인 유지

Day 16: 접속 시도
  → Refresh Token 만료
  → ❌ 로그인 필요
```

**결과**: 15일 동안 자동 로그인 유지

---

### 2. 2~3일에 한 번씩 접속하는 사용자

```
Day 1: 회원가입/로그인
  → Access Token (15분) + Refresh Token (15일) 발급

Day 3: 접속 (2일 후)
  → Access Token 만료됨 (15분 후 만료)
  → 페이지 로드 시 자동으로 Refresh Token으로 갱신
  → ✅ 자동 로그인 유지

Day 6: 접속 (3일 후)
  → Access Token 만료됨
  → 페이지 로드 시 자동으로 Refresh Token으로 갱신
  → ✅ 자동 로그인 유지

Day 10: 접속 (4일 후)
  → Access Token 만료됨
  → 페이지 로드 시 자동으로 Refresh Token으로 갱신
  → ✅ 자동 로그인 유지

Day 16: 접속 시도
  → Refresh Token 만료 (15일 지남)
  → ❌ 로그인 필요
```

**결과**: 15일 이내에 접속하면 자동 로그인 유지

---

### 3. 한 달에 한 번 접속하는 사용자

```
Day 1: 회원가입/로그인
  → Access Token (15분) + Refresh Token (15일) 발급

Day 30: 접속 시도
  → Refresh Token 만료 (15일 지남)
  → ❌ 로그인 필요
```

**결과**: 15일 후 접속 시 로그인 필요

---

### 4. 15일 이내에 여러 번 접속하는 사용자

```
Day 1: 회원가입/로그인
Day 2: 접속 → 자동 갱신 ✅
Day 5: 접속 → 자동 갱신 ✅
Day 8: 접속 → 자동 갱신 ✅
Day 12: 접속 → 자동 갱신 ✅
Day 14: 접속 → 자동 갱신 ✅
Day 16: 접속 시도 → Refresh Token 만료 ❌
```

**결과**: 15일 이내에 몇 번 접속하든 자동 로그인 유지

## 자동 갱신 동작 원리

### 페이지 로드 시 자동 갱신

1. **페이지 로드**
   ```
   사용자가 사이트 접속
   ```

2. **토큰 확인**
   ```
   App 컴포넌트에서 자동으로 확인:
   - Refresh Token이 있는가?
   - Access Token이 만료되었는가?
   ```

3. **자동 갱신**
   ```
   Access Token 만료 + Refresh Token 유효
   → 자동으로 새 Access Token 발급
   → 사용자는 아무것도 느끼지 못함
   ```

4. **결과**
   ```
   ✅ 자동 로그인 상태 유지
   ```

### API 요청 시 자동 갱신

1. **API 요청**
   ```
   사용자가 API 호출 (예: 데이터 조회)
   ```

2. **토큰 만료 감지**
   ```
   서버에서 Access Token 만료 확인
   → 401 에러 반환
   ```

3. **자동 재시도**
   ```
   프론트엔드에서 401 에러 감지
   → Refresh Token으로 새 Access Token 발급
   → 원래 요청 자동 재시도
   ```

4. **결과**
   ```
   ✅ 사용자는 에러를 보지 못함
   ✅ 자동으로 처리됨
   ```

## 요약

### ✅ 자동 로그인이 유지되는 경우

- **15일 이내**에 접속하는 경우
- **Access Token은 만료**되어도 **Refresh Token이 유효**하면 자동 갱신
- **접속 빈도와 무관** (매일이든 2~3일에 한 번이든)

### ❌ 로그인이 필요한 경우

- **15일 후** 접속하는 경우
- **Refresh Token이 만료**된 경우
- **로그아웃**한 경우
- **토큰이 삭제**된 경우

## 기술적 세부사항

### 토큰 만료 시간

- **Access Token**: 15분
  - 짧은 만료 시간으로 보안 강화
  - 만료되어도 자동 갱신

- **Refresh Token**: 15일
  - 긴 만료 시간으로 사용자 편의성 제공
  - 데이터베이스에 저장되어 관리

### 자동 갱신 트리거

1. **페이지 로드 시** (`App.tsx`)
   - 컴포넌트 마운트 시 자동 확인
   - 만료된 Access Token 자동 갱신

2. **API 요청 시** (향후 구현 가능)
   - 401 에러 발생 시 자동 재시도
   - Interceptor 패턴 사용

### 사용자 경험

- **자동 갱신**: 사용자가 아무것도 하지 않아도 자동으로 처리
- **투명한 동작**: 사용자는 토큰 갱신을 느끼지 못함
- **부드러운 전환**: 로그인 상태가 자연스럽게 유지됨
